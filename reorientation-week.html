Reorientation Week

Description: Reorientation Week is a first-person puzzle platformer which revolves around the core mechanic of changing the direction of gravity. The player is able to point their crosshair in any direction and, upon clicking, redirect gravity so that "down" is now the direction in which they clicked. In order to progress through the game, the player must solve puzzles and complete platforming segments which utilize this gravity-change mechanic.

Testing Adaptive Music System

I've implemented an audio system to the game which controls the levels of tracks based on whether or not gravity is currently altered. In order to test the system I composed a short loop which represents the general direction I'm thinking of going with the music. The loop is split into two tracks: a base layer which contains the instruments that will be heard during standard gameplay, and an additional layer which is overlaid while gravity has been reoriented.

Base Layer (Natural Gravity)

Additional Layer (Reoriented Gravity)

This track is mostly just for testing purposes, although it represents a few of the qualities which I'm aiming for:

  • The base layer should work well on its own, but should also work with the additional layer.
  • The base layer should generally be sparse and ambient, without any strong pulse.
  • The additional layer should introduce a more defined rhythm to the music.

When playing the game with this test track implemented, I did feel that the heavy synth bass was a bit cheesy. Since the player will likely be switching between natural gravity and reoriented gravity quite frequently, the instrumentation changes run the risk of being distracting or even annoying while playing. Moving forward I will try experimenting with more subtle layering.

April 4, 2019

Initial Thoughts and Planning

I first began working on Reorientation Week (still a working title) in 2018 as a project for my computer game design class. It was selected as one of the ten or so games that would be assigned a team to continue working on it for the following project. In this team I mostly focussed on coding and general directing. From the start, I had also intended to create a soundtrack for the game, however time constraints unfortunately meant I was only able to do some sound effect work. Now that I am returning to this project, I plan on redoing most of the sound effects, as well as creating multiple tracks for the game.

My first idea for the soundtrack in this game was do some experimentation with "adaptive music". Broadly speaking, this refers to music which dynamically changes in response to events occuring in-game. For Reorientation Week, I wanted to have the music be atmospheric and sparse during regular gameplay, but become more intense when the player has altered the direction of gravity. This could be accomplished, for example by introducing new instruments to the track, altering the harmonic content of the music, or increasing the tempo. The desired effect is that the sonic change reaffirms to the player the significance of the transformation. The music should also create tension as gravity can only remain altered for a short period of time.

The atmosphere of the game is pretty mysterious and otherworldly. I took a lot of inspiration from the shrines in The Legend of Zelda: Breath of the Wild. To capture the ethereal quality of the game I plan on making heavy use of synth sounds. My plan is to attempt to create most, if not all, the games music and sound effects using my Korg Prologue analog synthesizer.

The unique gameplay of Reorientation Week allows me to experiement with new ways of incorporating music into games. I'm super pumped to see how it turns out.
